Introdução

Versionamento de Código é o tratamento em versões do um código geradas por cada nova alteração feita.

Os Sistemas de Controle de Versão - VSC - são os responsáveis pelo controle destas versões de um arquivo ao longo do tempo; seja registrando o hstórico de atualizações, gerenciando alterações, organizando, controlando ou ainda dando segurança ao projeto.

Existem dois tipos de sistemas de controle de versão: os centralizados (CVCS) e os distribuídos (DVCS).

  • centralizado significa que temos um servidor central cujo banco de alterações é acessado por seus usuários e alterado diretamente nele;
  • distribuídos temos um servidor central que compartilha as versões localmente para cada usuário em sua máquina.

O Git é um sistema de controle de versão distribuído e gratuíto que usa um sistema de ramificações, os branches, e de fusões, os merges.

Breve Histórico

Em 2002 o projeto do núcleo do Linux (kernel) usava o DVCS proprietário Bitkeeper, numa relação cheia de conflitos, sendo a maior delas o fato do Linux ser código aberto (open source) e o Bitkeeper não. Em 2005 o Bitkeeper rescinde sua licença gratuita, o que faz com que a equipe liderada por Linus Torvalds, o criador do Linux e chefe do projeto, crie sua própria ferramenta, o Git. Feito em tempo recorde e mantendo-se gratuito.

Já o GitHub é uma plataforma de hospedagem de código para controle de versão com Git, e totalmente colaborativa. Criada em 2008 por quatro desenvolvedores, mas ganhou fama depois de resistir a um dos maiores ataques de DDoS (ataque distribuído de negação de serviço) do mundo. Foi comprada pela Microsoft em 2018 por US$ 7,5 bilões.
